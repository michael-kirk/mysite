Prerequisites to use this Document: test

You are a Service Desk analyst and have PUTTY & VNC installed, have received sufficient Linux training and are aware of the Store Passwords.

Software Version: None specific

Key:

Key Stroke or Button Press on Screen = <START>
OS Command = cd \
DB Command = select
Variable in DB command that must be changed = ‘AAA’, ‘BBB’, ‘CCC’

Description: Error shows when attempting to submit electronically


Root Cause: Client connection to the NHSEC directory has failed.

Problem Number: NA

Symptoms: Store are attempting to process a NHS Submission an error shows


Action Required: Mount the NHSEC directory using Rundeck

Solution:

1) Access the Rundeck Job using the URL below:-

https://rundeck.gg.specsavers.com/project/UKROI-Retail-Level1/job/show/a256c42d-bd13-4931-a547-02ef47b6ee36

2) When you have logged in and loaded Rundeck you will be presented with the following Job set-up screen: -

3) Enter the following:-

  • The 2 digit Client number for the Client they are getting the error message on
  • The 4 digit Store number for the target store
  • Select the target store from the drop-down menu

4) When entered correctly, press the green 'Run Job Now>' button

5) Once the job has run you should get the following message:-

Store Number Entered: XXXX

Client Number Entered: XX

Remounting /socrates

Remounting /socrates

Now ask store to test. If issue returns reboot client and test

6) If any other message is received, (other than 4 digit store number does not match) reboot the Client and attempt the fix again.

7) If the error is still unresolved following a reboot, attempt the fix using the manual steps listed below.

################################MANUAL PROCESS###############################

Errors that can be seen are;

"Error in electronic submission.

java.io.FileNotFoundException:

/socrates/nhsec/nhsec_DDDDDDDD_FFFF

F.xml (No such file or directory) " <DDDDDDDD = date & FFFFF = file name>

or

Permission Denied

To remedy that the client has not been setup to process NHS debtor submissions perform the following.

SSH / Putty to the client that has the fault/error message.

Confirm the redhat type of the client by running the below on the affected client :-

cat /etc/redhat-release

It should return either of the below results:

Red Hat Desktop release 4

Red Hat Enterprise Linux Client release 5.6

If RHEL 4 Run the below (Should the commands below for RHEL 4 not be recognised attempt the commands from RHEL 5.6 section)

umount //ice/socrates

mount //ice/socrates

cd /socrates/nhsec/;ls -l

If RHEL 5.6 run the below

umount /socrates

mount /socrates

cd /socrates/nhsec/;ls -l

you should now see similar to;

[root@client12 nhsec]# ls -l
total 16
drwxrwxrwx 1 root root 0 Apr 15 21:23 backupJunk
drwxrwxrwx 1 root root 0 Apr 18 21:23 backupReceived
drwxrwxrwx 1 root root 0 Apr 15 21:23 backupSent
drwxrwxrwx 1 root root 0 Apr 18 21:23 receivedEcf

Now ask store to test.

If issue remains try running the following to prompt the client to remount

mount -a

Failing this the last thing to attempt would be a client reboot to try and force the client to remount

If errors are seen as below if still an issue escalate to T2/IMTA for the below to be followed.

Step 4.

Putty to the Server and use the following command
vi /etc/exports

add the following to the file then save and exit the file
/export/socrates *(rw,sync,no_root_squash)

Run this command on the Server

exportfs -a

Connect to the Client affected, or use your already open live connection to the client and run the following command

vi /etc/fstab

Change the following line...

//ice/socrates /socrates cifs
username=guest,password=,dmask=777,fmask=777 0 0

to look like below then save and exit the file:
ice:/export/socrates /socrates nfs nouser,rw 0 0

Run commands

umount /socrates

mount -a


Now restart the client, load Socrates and ask store to test.
